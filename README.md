# 箏（琴）学習システム

## プロジェクト概要
小中高校の音楽授業で使用する箏（琴）の学習支援システム。  
生徒側と先生側で分かれた画面を持ち、課題の達成状況をブラウザのLocalStorageで管理する。

---

## 主要機能

### 生徒側（student.html）
| 機能 | 詳細 |
|------|------|
| 学年別課題タブ | ログイン学年に対応した課題一覧（スタンプラリー付き） |
| 曲別課題タブ | 海の声・島人ぬ宝・涙そうそう の3曲をタブ切替で表示 |
| 完了トグル | 各課題の完了/未完了をボタンで切替。LocalStorageに保存 |
| 進捗スタンプ | 達成数に応じてスタンプが変化するビジュアル進捗表示 |

### 先生側（teacher.html）
| タブ | 機能 |
|------|------|
| 進捗管理 | クラス別の生徒進捗をマトリクス表示・CSVエクスポート |
| 課題管理・CSV取込 | 学年別の課題をCSVでインポート（ドラッグ＆ドロップ対応） |
| **曲別課題管理** | 3曲それぞれの課題をCSVでインポート・一覧確認 |
| クラス設定 | 学年・組の有効化・追加・削除 |

---

## エントリーポイント（URL）

| パス | 説明 |
|------|------|
| `/` → `index.html` | 生徒ログイン画面 |
| `/student.html` | 生徒用学習システム（要ログイン） |
| `/teacher-login.html` | 先生ログイン画面 |
| `/teacher.html` | 先生用管理画面（要ログイン） |

---

## データモデル

### テーブル: `koto_tasks`（学年別課題）
| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | text | レコードID |
| grade | text | 対象学年（1/2/3） |
| task_number | number | 課題番号 |
| title | text | 課題名 |
| criteria | text | 達成基準 |
| is_active | bool | 有効フラグ |

### テーブル: `song_tasks`（曲別課題） ※新規追加
| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | text | レコードID |
| song_key | text | 曲識別キー（umi_no_koe / shimanchu_nu_takara / namida_sousou） |
| song_name | text | 曲名（表示用） |
| task_number | number | 課題番号 |
| title | text | 課題名 |
| criteria | text | 達成基準 |
| is_active | bool | 有効フラグ |

### テーブル: `school_settings`（クラス設定）
| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | text | レコードID |
| setting_key | text | キー名（grade_1_enabled, grade_1_classes 等） |
| setting_value | text | 値 |
| description | text | 説明 |

### LocalStorage
| キー | 内容 |
|------|------|
| `currentUser` | ログイン中のユーザー情報（JSON） |
| `userType` | ユーザー種別（student / teacher） |
| `koto_progress_{studentId}` | 学年別課題の達成状況（JSON） |
| `song_progress_{studentId}` | 曲別課題の達成状況（JSON） |

---

## 曲別課題フロー

1. **先生側** → 「曲別課題管理」タブ → 曲を選択 → CSVをアップロード
2. **生徒側** → ログイン（学年不問） → 「曲別課題」タブ → 曲タブを選択 → 課題確認・完了登録
